### Task 3: 接单时自动创建设计记录

**Files:**
- Modify: `src/app/api/orders/[id]/accept/route.ts`

当前逻辑：接单只更新订单状态
```javascript
const { data: order, error } = await supabase
  .from('orders')
  .update({ status: 'designing', ... })
  .eq('id', orderId)
  .select()
  .single()
```

新逻辑：接单成功后创建设计记录
```javascript
// 1. 创建设计记录
await adminSupabase.from('designs').insert({
  organization_id: order.organization_id,
  order_id: orderId,
  customer_id: order.customer_id,
  created_by: user.id,
  status: 'draft',
  title: `${order.customer_name} - 设计方案`,
})

// 2. 更新订单状态
const { data: order, error } = await supabase
  .from('orders')
  .update({ status: 'designing', ... })
  .eq('id', orderId)
  .select()
  .single()
```

## 风险和注意事项

1. **原子性**：接单时同时创建设计记录和更新订单状态，需要确保数据一致性
2. **历史数据**：已有设计方案没有 order_id，需要后续手动修复
3. **权限**：确保设计师只能看到分配给自己的订单
