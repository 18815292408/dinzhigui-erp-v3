# 仪表盘推进中订单下一步跳转修复实现计划

**Goal:** 修复仪表盘推进中订单表格中 `pending_shipment` 和 `in_install` 状态点击下一步跳转到不存在的安装单的问题。

**Architecture:** 将这两个状态的跳转目标从 `/installations` 改为 `/orders`，因为订单详情页已经为这两个阶段提供了完整的操作入口和正确的安装单链接。

**Tech Stack:** Next.js, TypeScript

---

## 实现任务

### Task 1: 修改 STAGE_META 的 href 指向

**Files:**
- Modify: `src/lib/dashboard-overview.ts`

- [ ] **Step 1: 修改 pending_shipment 的 href**

将 `pending_shipment` 的 `href` 从 `'/installations'` 改为 `'/orders'`：

```typescript
pending_shipment: { label: '待出货', nextAction: '填写出货/指派安装', href: '/orders' },
```

- [ ] **Step 2: 修改 in_install 的 href**

将 `in_install` 的 `href` 从 `'/installations'` 改为 `'/orders'`：

```typescript
in_install: { label: '安装中', nextAction: '更新安装进度', href: '/orders' },
```
