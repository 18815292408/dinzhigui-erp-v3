# 账号管理员工密码与资料修改功能设计

## 背景

当前 `/settings/users` 页面（账号管理）支持老板创建员工账号，但在员工离职或信息变更时，老板无法直接修改员工的密码或基本资料。这在紧急交接场景下非常不方便。

## 需求

1. **老板可以在账号管理页面修改员工的**：
   - 员工姓名（display_name）
   - 邮箱（email）
   - 手机号（phone）
   - 角色（role）
   - 登录密码（password）

2. **权限控制**：
   - 只有老板（role = owner）可以修改员工信息
   - 不能修改老板自己的信息
   - 只能修改同组织（organization_id）内的员工
   - 不能将员工角色改为 owner（避免越权）

3. **密码修改逻辑**：
   - 密码字段为可选，留空则不修改密码
   - 密码至少 6 位
   - 修改密码时使用 Supabase Auth Admin API (`auth.admin.updateUserById`)

4. **邮箱/手机号修改逻辑**：
   - 邮箱和手机号不能同时为空（至少保留一个）
   - 修改邮箱时需要同步更新 Supabase Auth 中的 email

## 现有代码分析

### 前端
- `src/app/(dashboard)/settings/users/page.tsx` — 账号管理页面，查询同组织用户列表
- `src/components/settings/user-list.tsx` — 用户列表组件，已有编辑弹窗 EditModal，目前仅支持修改角色和删除
- `src/components/settings/user-form.tsx` — 创建账号表单，字段与编辑类似

### 后端
- `src/app/api/users/route.ts` — POST 创建用户（owner/manager 可用）
- `src/app/api/admin/users/[id]/route.ts` — PATCH/DELETE 仅管理员可用，老板无法调用
- `/api/users/limits` — GET 获取角色创建限额

## 设计方案

### 1. 新增 API 路由：`PATCH /api/users/[id]`

供老板修改员工信息使用。

```typescript
PATCH /api/users/:id
Body: {
  display_name?: string
  email?: string | null
  phone?: string | null
  role?: string
  password?: string // optional, 留空不修改
}
```

权限校验：
1. 检查 session 存在
2. 检查 currentUser.role === 'owner'
3. 检查 targetUser.organization_id === currentUser.organization_id
4. 检查 targetUser.role !== 'owner'（不能改老板账号）
5. 检查新 role 不是 'owner'

修改逻辑：
1. 更新 `public.users` 表（display_name, email, phone, role）
2. 如果有 password，调用 `supabase.auth.admin.updateUserById(userId, { password })`
3. 如果 email 变更，同步更新 Supabase Auth 中的 email

### 2. 修改前端：`UserList` 编辑弹窗

扩展现有 `EditModal`，增加表单字段：
- 员工姓名（input）
- 邮箱（input）
- 手机号（input）
- 角色（select，排除 owner 选项）
- 新密码（input，password 类型，placeholder "留空则不修改"）

调用新的 `PATCH /api/users/[id]` 接口。

| 文件 | 变更类型 | 说明 |

| `src/app/api/users/[id]/route.ts` | 新增 | 老板修改员工信息 API |
