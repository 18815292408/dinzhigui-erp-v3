# 全屋定制门店智能 ERP 系统 — MVP 设计

## 1. 产品定位与目标

**产品定位**
全屋定制门店内部管理 ERP 系统，MVP 阶段服务单店、三类角色分工明确的中小型门店。

**核心目标**
验证市场需求：跑通全屋定制门店核心业务流程，1-2 个真实客户从录入到安装完成全流程贯通。

**与现有系统关系**
- 已存在：全屋定制报价系统、小红书获客系统
- MVP 阶段：**不接入**以上两个系统，独立验证
- 后续迭代：MVP 验证后逐步接入

---

## 2. 账号体系

**账号结构**：树形结构，老板为主账号

```
老板（主账号）
├── 导购账号 × ≤3
├── 设计师账号 × ≤3
└── 安装/售后账号 × ≤3
```

- **老板**：主账号，拥有系统所有功能，可创建/管理下属店员账号
- **导购/设计师/安装售后**：子账号，仅可见自己功能模块
- 每个店员对应一个独立登录账号（支持**邮箱或手机号**登录，二选一）
- 账号上限：1老板 + 9子账号（每类角色各最多3个），超出需额外付费

## 3. 用户角色与权限

| 角色 | 核心职责 | 可用功能模块 |
|------|---------|-------------|
| 老板 | 看数据、审核、账号管理 | 全部模块 + 账号管理 |
| 设计师 | 出方案、录系统 | 方案管理、客户查看 |
| 导购 | 谈客户、录意向、跟单 | 客户管理 |
| 安装/售后 | 查安装单、反馈安装结果 | 安装管理 |

**MVP 权限策略**：
- 老板可查看所有业务数据（全部客户、方案、安装单）
- 其他角色（导购/设计师/安装售后）之间数据暂不隔离
- 后续根据业务需要再扩展权限控制

---

## 4. 核心业务流程

```
[导购] → 录入客户信息
           ↓
      AI 意向分析 + 打标签（重度/中度/轻度意向）
           ↓
[设计师] → 接收客户需求，出具方案（人工录入系统）
           ↓
[安装/售后] → 接收安装单，执行并反馈结果
           ↓
[老板] → 查看数据看板，审核重要事项
```

## 5. 功能模块

### 5.1 客户管理（导购）
- 新增客户：姓名、联系方式、需求描述、房屋信息
- 意向跟进：记录沟通历史
- **AI 意向分析**：录入后自动分析，输出意向等级标签
- 客户列表：按意向标签筛选

### 5.2 方案管理（设计师）
- 创建方案：关联客户，录入设计方案（尺寸、户型、报价要素）
- 方案状态：草稿 / 已提交 / 已确认
- 方案列表：按状态筛选

### 5.3 安装管理（安装/售后）
- 安装单列表：查看待安装、进行中、已完成
- 安装单关联：每个安装单对应一个客户（及对应方案）
- 安装反馈：录入安装结果、异常情况
- 方案确认后：设计师确认方案 → 自动/手动创建安装单

### 5.4 数据看板（老板）
- 关键指标：新增客户数、意向分布、方案数量、安装完成率
- 图表展示：支持按时间筛选（今日/本周/本月）

### 5.5 权限管理（老板）
- 角色配置：设置店员账号和角色
- 数据隔离：确保老板数据与其他角色隔离

## 6. AI 能力设计

**唯一 AI 场景：客户意向分析**

| 项目 | 内容 |
|------|------|
| 触发时机 | 导购新建客户并保存后 |
| 调用方式 | DeepSeek API 直接调用 |
| 输入数据 | 客户基本信息 + 需求描述 |
| 输出 | 意向等级（重度/中度/轻度）+ 分析理由 |
| 展示位置 | 客户详情页标签区 |
| 调用失败处理 | 前端显示"分析中"状态，5秒内未返回则显示"暂未分析"，可手动重试 |
| 防抖规则 | 仅新建时触发一次，修改客户信息不重复调用AI |

**不做**：TTS 语音（文字转语音）、图像生成

**AI 扩展接口预留**：
- 语音转文字（STT）：导购/设计师谈单录音 → 转文字存为沟通凭证，预留接口，后续接入

## 7. 技术方案

| 项目 | 选择 |
|------|------|
| 应用形式 | Web App（响应式，适配 PC/iPad） |
| UI 风格 | 苹果设计语言（简洁、留白、一致性） |
| 后端 | Supabase（云端数据库 + 认证 + Edge Functions） |
| AI | DeepSeek API（直接调用，非 Supabase AI 插件） |
| 部署 | Vercel / Cloudflare Pages（静态部署） |

**数据库初步设计（后续迭代）**
- `customers` — 客户表
- `intentions` — AI 意向分析记录（关联 customers）
- `designs` — 方案表（关联 customers）
- `installations` — 安装单表（关联 customers，可选关联 designs）
- `users` — 用户表（含角色，关联 organization）
- `organizations` — 门店表（单店 MVP 固定一条记录）

## 8. MVP 不做的事

以下功能明确在 MVP 范围外：

- 支付 / 收款
- 库存 / 物流管理
- 多门店管理
- 客户主动联系导购（微信/小程序入口）
- 设计师 ↔ 导购 ↔ 安装 之间的数据隔离
- 报价系统接入
- 小红书获客系统接入
- TTS 语音
- 图像生成
- 自动化量房（人工录入方案数据）

---

## 9. 验收标准

**MVP 完成标准**
1. 1-2 个真实客户从录入到安装完成全流程跑通
2. 三类角色（导购、设计师、安装/售后）各自能完成核心操作
3. AI 意向分析能返回有效标签
4. 老板能看到数据看板
5. 旧系统（表格/微信）可停用，全面切换到新系统

---

## 10. MVP 开发优先级

| 优先级 | 功能 |
|--------|------|
| P0 | 客户录入 + AI 意向分析 |
| P0 | 安装单管理 + 反馈 |
| P0 | 老板数据看板 |
| P1 | 方案管理 |
| P1 | 用户角色配置（基础版） |
| P2 | 权限数据隔离（老板） |
