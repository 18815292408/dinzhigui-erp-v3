# 全屋定制ERP系统 - 流程重构设计

## 1. 产品定位

全屋定制门店内部管理ERP系统，服务于有设计师下单员合一（设计师同时负责下单）、独立售后岗的门店。

**核心变化**：从原有的「客户→方案→安装」简化为「订单流」统一管理，增加工厂拆分、打款、弹窗通知等核心功能。

---

## 2. 角色体系

| 角色 | 职责 | 数据可见范围 |
|------|------|------------|
| **老板** | 打款确认 + 月度统计 + 账号管理 | 所有数据（含财务金额） |
| **店长** | 日常运营管理 | 所有数据（财务金额隐藏） |
| **设计师** | 接单 → 选出图时间 → 出方案 → 下单工厂 → 填出货时间 | 只看被派单给自己的订单 |
| **销售** | 建档签单 → 选择设计师派单 | 只看自己建档的客户 |
| **安装/售后** | 跟进出货→到货→送货→安装→补件记录 | 只看被指派给自己的订单 |

**账号配置**：老板 × 1，店长 × 1，普通员工按需

---

## 3. 核心业务流程

```
[销售] 建档签单 → 选择设计师派单
    ↓ 弹窗通知设计师
[设计师] 接单（选择出图时间：7天/10天/12天/15天）
    ↓ 提交方案
[设计师] 下单至工厂（从预录入工厂列表选择，可拆分多家）
    ↓ 对单完成
[老板] 确认打款（弹窗通知）
    ↓
[设计师] 填写预计出货时间 + 选择安装售后人员
    ↓ 弹窗通知售后
[安装/售后] 接收订单 → 跟进全流程（出货→到货→送货→安装→补件）
    ↓ 安装完成
[归档] 按月存档（不可修改，只能添加备注）
```

## 4. 订单状态机

### 4.1 主状态

| 状态 | 状态值 | 说明 | 可执行动作 |
|------|--------|------|-----------|
| 待派单 | `pending_dispatch` | 销售已建档未派单 | 销售选择设计师派单 |
| 待接单 | `pending_design` | 设计师待接单 | 设计师接单（选出图时间） |
| 设计中 | `designing` | 设计师正在出方案 | 设计师提交方案 |
| 待下单 | `pending_order` | 方案已确认待下单 | 设计师下单至工厂 |
| 待打款 | `pending_payment` | 工厂已下单待打款 | 老板确认打款 |
| 待出货 | `pending_shipment` | 已打款待出货 | 设计师填写出货时间 |
| 安装中 | `in_install` | 售后跟进安装 | 售后更新进度 |
| 已完结 | `completed` | 全部流程完成 | 归档，只能添加备注 |

### 4.2 安装子状态

当订单主状态为 `in_install` 时，使用以下子状态跟进安装进度：

| 子状态 | 状态值 | 说明 | 可执行动作 |
|--------|--------|------|-----------|
| 待出货 | `pending_ship` | 等待工厂出货 | 售后确认出货 |
| 已出货 | `shipped` | 工厂已发货 | 售后确认到货 |
| 已到货 | `arrived` | 货物已到达 | 售后安排送货 |
| 送货中 | `delivering` | 正在送货 | 售后确认送达 |
| 安装中 | `installing` | 正在安装 | 安装人员更新进度 |
| 补件中 | `supplement_pending` | 有补件待处理 | 售后发起补件 |
| 安装完成 | `installed` | 安装全部完成 | 订单完结归档 |

**补件流程**：
1. 售后/安装账号发现需要补件 → 发起补件申请
2. 系统弹窗通知设计师（紧急）："有订单需要补件，请立即处理"
3. 设计师确认补件 → 继续安装流程
4. 补件完成后 → 更新安装进度

### 4.3 超期判断标准

| 超期类型 | 判断标准 |
|---------|---------|
| 出图超时 | 当前时间 > design_due_date |
| 待打款超期 | 对单完成超过3天未打款 |
| 安装延误 | 当前时间 > estimated_shipment_date + 3天仍未出货 |

## 5. 弹窗通知机制

### 5.1 通知优先级

| 优先级 | 颜色 | 触发场景 | 通知对象 |
|--------|------|---------|---------|
| 🔴 紧急 | 红色 | 新订单派发、出图超时、待打款超期、安装延误、补件申请 | 相关负责人 |
| 🟠 重要 | 橙色 | 提前1天出图提醒、提前7天出货提醒 | 设计师 + 派单销售 |
| 🔵 一般 | 蓝色 | 待处理订单、方案已确认等 | 相关负责人 |
| 🟣 信息 | 紫色 | 安装完成、订单归档等 | 相关负责人 |

**通知规则**：
- **出图提醒**：同时通知设计师和派单的销售（导购）
- **其他事项**：只通知直接负责人

### 5.2 弹窗交互

**所有弹窗统一内容格式**：
- 标题：根据优先级显示「紧急事项」「重要提醒」「新消息」
- 内容：根据优先级显示不同颜色提示条 + 「您有事项需要处理，请立即前往消息中心」
- 按钮：【知道了】【立即前往】

**交互规则**：
- 必须点击按钮才能关闭弹窗
- 点击「立即前往」跳转到消息中心
- 点击「知道了」关闭弹窗但事项仍在消息中心

### 5.3 消息中心

- 入口：顶部导航栏铃铛图标
- 展示：所有待处理事项列表
- 排序：按优先级 + 时间排序
- 每条消息显示：类型图标 + 标题 + 摘要 + 时间 + 快捷操作按钮

**消息权限矩阵**：

| 角色 | 可查看范围 |
|------|-----------|
| 老板 | 所有消息（全员） |
| 店长 | 所有消息（全员） |
| 设计师 | 只看自己的消息 |
| 销售 | 只看自己的消息 |
| 安装/售后 | 只看自己的消息 |

## 6. 数据表设计

### 6.1 orders 订单主表

```sql
CREATE TABLE orders (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  organization_id UUID REFERENCES organizations(id) NOT NULL,
  order_no TEXT NOT NULL UNIQUE,  -- 格式：DD-YYYYMMDD-NNN
  customer_name TEXT NOT NULL,
  customer_phone TEXT,
  customer_address TEXT,
  house_type TEXT,  -- 户型
  house_area DECIMAL(10,2),  -- 面积
  created_by UUID REFERENCES users(id),  -- 销售
  assigned_designer UUID REFERENCES users(id),  -- 设计师
  assigned_installer UUID REFERENCES users(id),  -- 安装售后
  status TEXT NOT NULL DEFAULT 'pending_dispatch',
  design_due_days INTEGER,  -- 出图时间选择：7/10/12/15
  design_due_date DATE,  -- 预计出图日期
  factory_records JSONB DEFAULT '[]',  -- 工厂拆分记录
  payment_status TEXT DEFAULT 'unpaid',  -- unpaid/paid（按订单整体）
  payment_confirmed_at TIMESTAMPTZ,  -- 打款确认时间
  estimated_shipment_date DATE,  -- 预计出货日期
  installation_status TEXT DEFAULT 'pending_ship',  -- 安装子状态
  completed_at TIMESTAMPTZ,
  archived_at TIMESTAMPTZ,
  remarks JSONB DEFAULT '[]',  -- 完结后备注
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

### 6.2 factories 工厂表

```sql
CREATE TABLE factories (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  organization_id UUID REFERENCES organizations(id) NOT NULL,
  name TEXT NOT NULL,  -- 工厂名称
  contact_name TEXT,
  contact_phone TEXT,
  address TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### 6.3 notifications 消息通知表

```sql
CREATE TABLE notifications (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  organization_id UUID REFERENCES organizations(id) NOT NULL,
  user_id UUID REFERENCES users(id) NOT NULL,  -- 接收人
  type TEXT NOT NULL,  -- new_order/overtime/payment/install 等
  priority TEXT NOT NULL,  -- urgent/important/normal/info
  title TEXT NOT NULL,
  summary TEXT,
  related_order_id UUID REFERENCES orders(id),
  is_read BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### 6.4 order_factory_records 工厂拆分记录（JSONB内嵌）

```json
[
  {
    "factory_id": "uuid",
    "factory_name": "A厂",
    "contact_name": "张三",
    "contact_phone": "138xxxx",
    "amount": 68000
  }
]
```

**说明**：打款按订单整体操作，无需区分每个工厂是否已打款。

### 6.5 installations 安装记录（增强）

```sql
ALTER TABLE installations ADD COLUMN shipment_date DATE;
ALTER TABLE installations ADD COLUMN arrival_date DATE;
ALTER TABLE installations ADD COLUMN delivery_date DATE;
ALTER TABLE installations ADD COLUMN install_date DATE;
ALTER TABLE installations ADD COLUMN installer_name TEXT;
ALTER TABLE installations ADD COLUMN supplement_records JSONB DEFAULT '[]';  -- 补件记录
```

### 6.6 supplements 补件记录表

```sql
CREATE TABLE supplements (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  organization_id UUID REFERENCES organizations(id) NOT NULL,
  order_id UUID REFERENCES orders(id) NOT NULL,
  created_by UUID REFERENCES users(id) NOT NULL,  -- 发起人（售后/安装）
  description TEXT NOT NULL,  -- 补件原因/内容
  status TEXT NOT NULL DEFAULT 'pending',  -- pending/confirmed/completed
  confirmed_by UUID REFERENCES users(id),  -- 确认补件的设计师
  confirmed_at TIMESTAMPTZ,
  completed_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

**简化追踪**：只记录补件内容、发起时间、完成时间，不追踪物流节点。

## 7. 功能模块

### 7.1 订单管理

| 功能 | 说明 |
|------|------|
| 创建订单 | 销售建档时创建，填写客户信息 |
| 派单 | 销售选择设计师，弹窗通知设计师 |
| 接单 | 设计师接单，选择出图时间（7/10/12/15天） |
| 提交方案 | 设计师提交方案，进入下单环节 |
| 下单工厂 | 设计师从预录入工厂选择，可拆多家 |
| 确认打款 | 老板操作，弹窗通知设计师 |
| 填写出货时间 | 设计师填写 + 选择安装售后 |
| 安装跟进 | 售后更新安装子状态：出货→到货→送货→安装 |
| 补件申请 | 售后发起补件 → 弹窗通知设计师 → 设计师确认 |
| 完结归档 | 订单完结，按月存档，不可修改 |
| 添加备注 | 完结后只能添加备注 |

### 7.2 工厂管理

| 功能 | 说明 |
|------|------|
| 预录入工厂 | 老板/店长录入工厂信息 |
| 工厂列表 | 设计师下单时从列表选择 |

### 7.3 消息中心

| 功能 | 说明 |
|------|------|
| 弹窗通知 | 紧急事项弹窗 + 铃铛红点 |
| 消息列表 | 按优先级排序，支持快捷操作 |
| 已读标记 | 查看后标记已读 |
| 补件通知 | 售后发起补件 → 弹窗通知设计师 |

### 7.4 月度统计（老板视角）

| 维度 | 数据 |
|------|------|
| 销售统计 | 每月签单数、收款金额及明细 |
| 设计师统计 | 每月下单数量、订单总金额 |

## 8. 订单编号规则

格式：`DD-YYYYMMDD-NNN`

示例：`DD-20260421-001`

- DD：固定前缀
- YYYYMMDD：日期
- NNN：当日序号，不足3位补零

---

## 9. 完结归档规则

| 规则 | 说明 |
|------|------|
| 完结条件 | 安装完成 |
| 归档维度 | 按月归档（YYYY-MM） |
| 修改限制 | 完结后不可修改订单主体 |
| 可添加 | 可添加备注条目 |
