# 客户管理列表查询修复设计

## 问题

当前客户管理使用 `has_active_order` 字段来区分"订单创建"和"订单跟进"，但该字段在派单和安装完成时没有被正确更新，导致数据不一致。

部分客户（如1111、你们好、zy）有设计记录但在 orders 表没有记录，导致显示错误。

## 解决方案

改用 `orders` 和 `designs` 表作为权威数据源，直接计算客户的状态。

## 查询逻辑

### 订单创建
客户**从未**在 orders 表和 designs 表中出现（没有任何订单或设计记录）

### 订单跟进
客户满足以下任一条件：
- 在 orders 表中有**非 completed** 状态的订单
- 在 designs 表中有**进行中**的设计（draft, submitted）

## 实现

```typescript
// 获取所有订单和设计
const { data: allOrders } = await adminSupabase
  .from('orders')
  .select('customer_name, status')
  .eq('organization_id', user.organization_id)

const { data: allDesigns } = await adminSupabase
  .from('designs')
  .select('customer_id, status')
  .eq('organization_id', user.organization_id)
  .in('status', ['draft', 'submitted'])

// 有进行中订单的客户名称
const customerNamesWithActiveOrders = new Set(
  (allOrders || [])
    .filter(o => o.status !== 'completed')
    .map(o => o.customer_name)
    .filter(Boolean)
)

// 有进行中设计的客户ID
const customerIdsWithActiveDesigns = new Set(
  (allDesigns || [])
    .filter(d => ['draft', 'submitted'].includes(d.status))
    .map(d => d.customer_id)
    .filter(Boolean)
)

// 订单跟进：满足任一条件
const isInFollowup = c => 
  customerNamesWithActiveOrders.has(c.name) ||
  customerIdsWithActiveDesigns.has(c.id)

// 订单创建：既没有订单也没有设计
const customersWithoutOrders = allCustomers.filter(c => !isInFollowup(c))
const customersWithOrders = allCustomers.filter(c => isInFollowup(c))
```
