# 客户管理列表查询修复设计

## 问题

当前客户管理使用 `has_active_order` 字段来区分"订单创建"和"订单跟进"，但该字段在派单和安装完成时没有被正确更新，导致数据不一致。

## 解决方案

改用 `orders` 表作为权威数据源，直接计算客户的订单状态。

## 查询逻辑

### 订单创建
客户**从未**在 orders 表中出现（没有任何订单记录）

```typescript
// 获取从未有过订单的客户
const { data: allCustomers } = await adminSupabase
  .from('customers')
  .select('*')
  .eq('organization_id', user.organization_id)

const { data: allOrders } = await adminSupabase
  .from('orders')
  .select('customer_name')
  .eq('organization_id', user.organization_id)

const customerNamesWithOrders = new Set(
  (allOrders || [])
    .map(o => o.customer_name)
    .filter(Boolean)
)

const customersWithoutOrders = (allCustomers || []).filter(
  c => !customerNamesWithOrders.has(c.name)
)
```

### 订单跟进
客户在 orders 表中且有**非 completed** 状态的订单

```typescript
const { data: activeOrders } = await adminSupabase
  .from('orders')
  .select('customer_name, status, order_no, signed_amount')
  .eq('organization_id', user.organization_id)
  .neq('status', 'completed')

// 获取有进行中订单的客户名称
const activeCustomerNames = [...new Set((activeOrders || []).map(o => o.customer_name).filter(Boolean))]

// 获取这些客户的详情
const { data: customersWithActiveOrders } = activeCustomerNames.length > 0
  ? await adminSupabase
      .from('customers')
      .select('*, orders(customer_name, status, order_no, signed_amount)')
      .eq('organization_id', user.organization_id)
      .in('name', activeCustomerNames)
  : { data: [] }
```

## 优点

1. **数据准确** - 始终基于 orders 表计算，不会因 API bug 导致不一致
2. **无需维护** - 安装完成时不需要更新客户状态
3. **逻辑清晰** - "订单创建"=无订单，"订单跟进"=有进行中订单
