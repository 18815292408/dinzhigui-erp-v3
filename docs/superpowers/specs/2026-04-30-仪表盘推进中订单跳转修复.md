# 仪表盘推进中订单下一步跳转修复

## 问题

仪表盘"推进中订单"表格中，订单状态为 `pending_shipment`（待出货）时，点击"下一步"（填写出货/指派安装）跳转到 `/installations/<order.id>`。

但此时安装单可能尚未创建（安装单是在"指派安装"或"确认出货"操作中才创建的），导致安装详情页找不到对应记录，显示"安装单不存在"。

同样的问题也存在于 `in_install`（安装中），用订单 ID 去查安装单 ID，两者 ID 不同。

## 根因

`STAGE_META` 中两个状态的 `href` 指向 `/installations`，而表格中链接构造为 `/installations/<order.id>`，用订单 ID 查找安装单，必然不匹配。

```typescript
// dashboard-overview.ts
pending_shipment: { label: '待出货', nextAction: '填写出货/指派安装', href: '/installations' },
in_install: { label: '安装中', nextAction: '更新安装进度', href: '/installations' },
```

而订单详情页（`/orders/[id]`）已经为 `pending_shipment` 和 `in_install` 两个阶段提供了对应的操作卡片，其中正确使用了安装单 ID 来跳转。

## 修复

将 `pending_shipment` 和 `in_install` 两个状态的 `href` 从 `/installations` 改为 `/orders`。

订单详情页已有完整操作入口，用户可以：
- `pending_shipment`：在订单页指派安装/填写出货
- `in_install`：在订单页点击安装单链接跳转到正确安装页

## 变更文件

- `src/lib/dashboard-overview.ts` — 修改 STAGE_META 中两个 entry 的 href
