# 客户管理订单状态分离设计

## 1. 概述

将客户管理分为两个状态：**订单创建**和**订单跟进**，实现订单流转的可视化管理。

---

## 2. 数据模型变更

### 2.1 orders 表新增字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `signed_amount` | DECIMAL(12,2) | 签单金额（移交设计时填写） |
| `final_order_amount` | DECIMAL(12,2) | 最终下单销售额（设计师下单前填写） |

### 2.2 customers 表新增字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `order_stage` | TEXT | 当前订单所处阶段 |
| `has_active_order` | BOOLEAN | 是否有进行中订单 |

---

## 3. 客户管理状态定义

### 3.1 订单创建（无进行中订单）

**触发条件：** 客户无 `has_active_order` 或 `has_active_order = false`

**显示内容：**
- 客户基本信息（姓名、电话、房型）
- 意向等级
- 跟进状态

**可执行操作：**
- 新建订单（转交设计）
- 编辑客户信息
- 添加跟进记录

### 3.2 订单跟进（有进行中订单）

**触发条件：** `has_active_order = true`

**显示内容：**
- 客户详细信息
- 签单金额（signed_amount）
- 订单当前阶段（order_stage）
- 订单编号

**可执行操作：**
- 查看订单详情
- 更新跟进记录

---

## 4. 业务流程

### 4.1 移交设计流程

```
销售选择设计师 → 填写签单金额 → 创建订单 → 通知设计师
```

**实现逻辑：**
1. 转交设计弹窗增加"签单金额"必填字段
2. 创建订单时记录 signed_amount
3. 更新客户的 has_active_order = true，order_stage = 'pending_design'
4. 订单派发给设计师，通知设计师接单

### 4.2 设计师接单流程

```
设计师接单 → 选择出图时间 → 进入设计中 → 提交方案 → 填写最终下单销售额 → 下单工厂
```

**实现逻辑：**
1. 设计师接单后，订单状态变为 `designing`
2. 客户 order_stage 更新为 'designing'
3. 设计师提交方案前，必须填写 final_order_amount
4. 如果签单金额为空，系统弹窗提醒导购填写

### 4.3 订单完结归档

```
安装完成 → 订单完结 → 客户 has_active_order = false
```

**实现逻辑：**
1. 订单状态变为 `completed`
2. 客户的 has_active_order = false，order_stage 保留归档状态
3. 客户重新回到"订单创建"状态

---

## 5. 下单门槛验证

### 5.1 设计师下单工厂前检查

**前置条件：** 订单状态为 `pending_order`

**检查逻辑：**
```
if (signed_amount == null || signed_amount == 0) {
  弹窗提醒： "签单金额未填写，请通知导购填写后再下单"
  阻止下单操作
}
```

**通知机制：**
- 弹窗显示：签单金额未填写
- 自动发送消息通知导购
- 导购填写后，设计师重新下单

---

## 6. 客户管理页面改造

### 6.1 列表页改造

**现状：** 统一显示所有客户

**改造后：**
- 默认显示"订单创建"状态的客户
- Tab 切换：订单创建 | 订单跟进
- 订单跟进 Tab 显示有进行中订单的客户

### 6.2 订单跟进详情显示

**显示字段：**
- 客户基本信息
- 签单金额（signed_amount）
- 最终下单销售额（final_order_amount）
- 当前订单阶段（order_stage）
- 订单编号

---

## 7. API 变更

### 7.1 创建订单 API (`POST /api/orders`)

**新增字段：**
- `signed_amount` - 必填

### 7.2 转交设计 API (`POST /api/orders/[id]/dispatch`)

**变更：**
- 接收 `signed_amount` 参数
- 更新客户的 has_active_order 和 order_stage

### 7.3 下单工厂 API (`POST /api/orders/[id]/place-order`)

**新增检查：**
- 验证 signed_amount 不为空

### 7.4 提交方案 API (`POST /api/orders/[id]/submit-design`)

**新增字段：**
- `final_order_amount` - 必填

---

## 8. 消息通知

### 8.1 新增通知类型

| 通知类型 | 触发场景 | 接收人 | 优先级 |
|---------|---------|--------|--------|
| `signed_amount_required` | 设计师下单时未填签单金额 | 导购 | 紧急 |

---

## 9. 数据库迁移

```sql
-- orders 表新增字段
ALTER TABLE orders ADD COLUMN signed_amount DECIMAL(12,2);
ALTER TABLE orders ADD COLUMN final_order_amount DECIMAL(12,2);

-- customers 表新增字段
ALTER TABLE customers ADD COLUMN order_stage TEXT DEFAULT NULL;
ALTER TABLE customers ADD COLUMN has_active_order BOOLEAN DEFAULT FALSE;
```
